Vue의 선언적 렌더링 모델은 대부분의 직접적인 DOM 조작을 추상화해주지만, 여전히 기본 DOM 요소에 직접 접근해야 하는 경우가 있을 수 있다. 이를 위해 특별한 `ref` 속성을 사용할 수 있다.
```vue
<input ref="input">
```
`ref`는 `v-for` 챕터에서 다룬 `key` 속성과 유사한 특별한 속성이다. 이를 통해 마운트된 후 특정 DOM 요소나 자식 컴포넌트 인스턴스에 직접 참조를 얻을 수 있다. 예를 들어, 컴포넌트가 마운트될 때 프로그래밍적으로 input에 포커스를 주거나, 요소에 3rd party 라이브러리를 초기화할 때 유용할 수 있다.

### ref 접근하기

Composition API에서 참조를 얻으려면 [`useTemplateRef()`](https://ko.vuejs.org/api/composition-api-helpers.html#usetemplateref)  헬퍼를 사용할 수 있다.
```vue
<script setup>
import { useTemplateRef, onMounted } from 'vue'

// 첫 번째 인자는 템플릿의 ref 값과 일치해야 합니다
const input = useTemplateRef('my-input')

onMounted(() => {
  input.value.focus()
})
</script>

<template>
  <input ref="my-input" />
</template>
```

TypeScript를 사용할 때, Vue의 IDE 지원과 `vue-tsc`는 일치하는 `ref` 속성이 사용된 요소나 컴포넌트에 따라 `input.value`의 타입을 자동으로 추론한다.

#### 3.5 이전 버전에서의 사용법

`useTemplateRef()`가 도입되지 않은 3.5 이전 버전에서는, 템플릿 ref 속성의 값과 일치하는 이름으로 ref를 선언해야 한다.
```vue
<script setup>
import { ref, onMounted } from 'vue'

// 요소 참조를 저장할 ref를 선언합니다
// 이름은 템플릿 ref 값과 일치해야 합니다
const input = ref(null)

onMounted(() => {
  input.value.focus()
})
</script>

<template>
  <input ref="input" />
</template>
```

`<script setup>`을 사용하지 않는 경우, `setup()`에서 ref를 반환해야 한다.

```js
export default {
  setup() {
    const input = ref(null)
    // ...
    return {
      input
    }
  }
}
```
ref는 **컴포넌트가 마운트된 후에만** 접근할 수 있다는 점에 유의하라. 템플릿 표현식에서 `input`에 접근하려고 하면, 첫 렌더에서는 `null`이 된다. 이는 요소가 첫 렌더 이후에야 존재하기 때문이다.

템플릿 ref의 변화를 감시하려는 경우, ref가 `null` 값을 가질 수 있는 상황을 반드시 고려해야 한다.
```js
watchEffect(() => {
  if (input.value) {
    input.value.focus()
  } else {
    // 아직 마운트되지 않았거나, 요소가 언마운트됨 (예: v-if에 의해)
  }
})
```

### 컴포넌트에 ref 사용하기

> 이 섹션은 [컴포넌트](https://ko.vuejs.org/guide/essentials/component-basics.html)에 대한 지식을 전제로 한다.

### `v-for` 내부의 ref (3.5+)

`v-for` 내부에서 `ref`를 사용할 때, 해당 ref는 Array 값을 가져야 하며, 마운트 후 요소들로 채워진다.
```vue
<script setup>
import { ref, useTemplateRef, onMounted } from 'vue'

const list = ref([
  /* ... */
])

const itemRefs = useTemplateRef('items')

onMounted(() => console.log(itemRefs.value))
</script>

<template>
  <ul>
    <li v-for="item in list" ref="items">
      {{ item }}
    </li>
  </ul>
</template>
```

#### 3.5 이전 버전에서의 사용법

`useTemplateRef()`가 도입되지 않은 3.5 이전 버전에서는, 템플릿 ref 속성의 값과 일치하는 이름으로 ref를 선언해야 하며, ref는 배열 값을 가져야 한다.
```vue
<script setup>
import { ref, onMounted } from 'vue'

const list = ref([
  /* ... */
])

const itemRefs = ref([])

onMounted(() => console.log(itemRefs.value))
</script>

<template>
  <ul>
    <li v-for="item in list" ref="itemRefs">
      {{ item }}
    </li>
  </ul>
</template>
```
ref 배열은 소스 배열과 **동일한 순서를 보장하지 않는다**는 점에 유의해야 한다.

### 함수 ref
문자열 키 대신, `ref` 속성은 함수에 바인딩할 수도 있다. 이 함수는 각 컴포넌트 업데이트 시 호출되며, 요소 참조를 어디에 저장할지 완전히 자유롭게 결정할 수 있다. 함수는 첫 번째 인자로 요소 참조를 받는다.
```vue
<input :ref="(el) => { /* el을 속성이나 ref에 할당 */ }">
```
함수 대신 ref 이름 문자열을 전달하는 것이 아니라, 동적 `:ref` 바인딩을 사용하고 있다. **요소가 언마운트될 때 인자는 `null`이 된다**. 물론, 인라인 함수 대신 메서드를 사용할 수도 있다.
