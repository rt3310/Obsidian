### `v-for`
`v-for` 디렉티브를 사용하여 배열을 기반으로 항목 목록을 렌더링할 수 있다.
`v-for` 디렉티브는 `item in items` 형태의 특별한 문법을 필요로 하며, 여기서 `items`는 소스 데이터 배열이고 `item`은 반복되는 배열 요소의 **별칭**이다.

```js
const items = ref([{ message: 'Foo' }, { message: 'Bar' }])
```
```vue
<li v-for="item in items">
  {{ item.message }}
</li>
```

`v-for`의 스코프 내에서는 템플릿 표현식이 모든 부모 스코프 속성에 접근할 수 있다. 또한, `v-for`는 현재 항목의 인덱스를 위한 두 번째 별칭도 지원한다.
```js
const parentMessage = ref('Parent')
const items = ref([{ message: 'Foo' }, { message: 'Bar' }])
```
```vue
<li v-for="(item, index) in items">
  {{ parentMessage }} - {{ index }} - {{ item.message }}
</li>
```

`v-for`의 변수 스코프는 다음 JavaScript와 유사하다.
```js
const parentMessage = 'Parent'
const items = [
  /* ... */
]

items.forEach((item, index) => {
  // 외부 스코프의 `parentMessage`에 접근 가능
  // 하지만 `item`과 `index`는 여기서만 사용 가능
  console.log(parentMessage, item.message, index)
})
```

`v-for` 값이 `forEach` 콜백의 함수 시그니처와 일치하는 것을 볼 수 있다. 실제로, 함수 인자 구조 분해와 유사하게 `v-for`의 항목 별칭에서도 구조 분해를 사용할 수 있다.
```vue
<li v-for="{ message } in items">
  {{ message }}
</li>

<!-- 인덱스 별칭과 함께 -->
<li v-for="({ message }, index) in items">
  {{ message }} {{ index }}
</li>
```

중첩된 `v-for`의 경우, 스코프는 중첩 함수와 유사하게 동작한다. 각 `v-for` 스코프는 부모 스코프에 접근할 수 있다.
```vue
<li v-for="item in items">
  <span v-for="childItem in item.children">
    {{ item.message }} {{ childItem }}
  </span>
</li>
```

또한, 구분자로 `in` 대신 `of`를 사용할 수도 있어, JavaScript의 반복자 문법과 더 유사하게 만들 수 있다.
```vue
<div v-for="item of items"></div>
```

### 객체에 대한 `v-for`
`v-for`를 사용하여 객체의 속성을 반복할 수도 있다. 반복 순서는 객체에 대해 `Object.values()`를 호출한 결과를 기반으로 한다.
```js
const myObject = reactive({
  title: 'How to do lists in Vue',
  author: 'Jane Doe',
  publishedAt: '2016-04-10'
})
```
```vue
<ul>
  <li v-for="value in myObject">
    {{ value }}
  </li>
</ul>
```

속성의 이름(즉, 키)에 대한 두 번째 별칭도 제공할 수 있다.
```vue
<li v-for="(value, key) in myObject">
  {{ key }}: {{ value }}
</li>
```

그리고 인덱스에 대한 또 다른 별칭도 사용할 수 있다.
```vue
<li v-for="(value, key, index) in myObject">
  {{ index }}. {{ key }}: {{ value }}
</li>
```

### 범위에 대한 `v-for`
`v-for`는 정수도 받을 수 있다. 이 경우 `1...n` 범위에 따라 해당 횟수만큼 템플릿을 반복한다.

```vue
<span v-for="n in 10">{{ n }}</span>
```
여기서 `n`은 0이 아닌 **1부터 시작**한다는 점에 유의하라.

### `<template>`에서의 `v-for`
템플릿 `v-if`와 유사하게, 여러 요소의 블록을 렌더링하기 위해 `<template>` 태그와 함께 `v-for`를 사용할 수도 있다. 예를 들어,
```vue
<ul>
  <template v-for="item in items">
    <li>{{ item.msg }}</li>
    <li class="divider" role="presentation"></li>
  </template>
</ul>
```

### `v-if`와 함께 사용하는 `v-for`
동일한 노드에 존재할 때, `v-if`가 `v-for`보다 우선순위가 높다. 즉, `v-if` 조건은 `v-for`의 스코프 변수에 접근할 수 없다.
```vue
<!--
이 코드는 "todo" 속성이 인스턴스에 정의되어 있지 않기 때문에
오류가 발생합니다.
-->
<li v-for="todo in todos" v-if="!todo.isComplete">
  {{ todo.name }}
</li>
```

이 문제는 `v-for`를 감싸는 `<template>` 태그로 옮겨서 해결할 수 있다(이 방법이 더 명확하기도 하다).
```vue
<template v-for="todo in todos">
  <li v-if="!todo.isComplete">
    {{ todo.name }}
  </li>
</template>
```

> [!warning]
> 암묵적인 우선순위 때문에 `v-if`와 `v-for`를 동일한 요소에 사용하는 것은 **권장되지 않는다**.
> 
> 이렇게 하고 싶어지는 일반적인 두 가지 경우가 있다.
> - 리스트에서 항목을 필터링할 때(예: `v-for="user in users" v-if="user.isActive"`). 이 경우, `users`를 필터링된 리스트(예: `activeUsers`)를 반환하는 새로운 계산 속성으로 대체하라.
> - 리스트가 숨겨져야 할 때 렌더링을 피하고 싶을 때(예: `v-for="user in users" v-if="shouldShowUsers"`). 이 경우, `v-if`를 컨테이너 요소(예: `ul`, `ol`)로 옮겨라.


### `key`로 상태 유지하기
Vue가 `v-for`로 렌더링된 요소 목록을 업데이트할 때, 기본적으로 "제자리 패치(in-place patch)" 전략을 사용한다.
**데이터 항목의 순서가 변경된 경우, Vue는 DOM 요소를 항목 순서에 맞게 이동시키는 대신 각 요소를 제자리에 패치하여 해당 인덱스에 렌더링되어야 할 내용을 반영한다**.

이 기본 모드는 효율적이지만, **리스트 렌더 출력이 자식 컴포넌트 상태나 임시 DOM 상태(예: 폼 입력 값)에 의존하지 않을 때만 적합하다**.

Vue가 각 노드의 정체성을 추적하고, 기존 요소를 재사용 및 재정렬할 수 있도록 힌트를 주려면 각 항목에 고유한 `key` 속성을 제공해야 한다.
```vue
<div v-for="item in items" :key="item.id">
  <!-- 내용 -->
</div>
```

`<template v-for>`를 사용할 때는 `key`를 `<template>` 컨테이너에 지정해야 한다.
```vue
<template v-for="todo in todos" :key="todo.name">
  <li>{{ todo.name }}</li>
</template>
```

> [!warning]
> 여기서 `key`는 `v-bind`로 바인딩되는 특별한 속성이다. [객체에 대한 `v-for` 사용](https://ko.vuejs.org/guide/essentials/list.html#v-for-with-an-object)에서의 속성 키 변수와 혼동하지 말 것.

가능하다면, 반복되는 DOM 콘텐츠가 단순한 경우(즉, 컴포넌트나 상태를 가진 DOM 요소가 없는 경우)를 제외하고, 또는 성능 향상을 위해 기본 동작에 의존하려는 특별한 경우가 아니라면, `v-for`와 함께 항상 `key` 속성을 제공하는 것이 좋다.

`key` 바인딩은 원시 값(즉, 문자열과 숫자)을 기대한다. 객체를 `v-for`의 키로 사용하지 마라. `key` 속성의 자세한 사용법은 [`key` API 문서](https://ko.vuejs.org/api/built-in-special-attributes.html#key)를 참고.
